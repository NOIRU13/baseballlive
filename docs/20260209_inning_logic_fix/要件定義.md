# 要件定義：イニング加算ロジックの修正

## 1. 背景

`admin-simple.html` からの操作において、ゴロ（groundout）を入力した際にイニング（アウト数）が 0.2（2アウト分）加算されてしまう不具合が発生している。これは野球のルールおよびユーザーの期待に反する挙動である。

## 2. 目的

- ゴロおよび通常のアウト時に、アウトカウントが正しく 1 つ（0.1）だけ増えるように修正する。
- 併殺（ダブルプレー）時には、アウトカウントが正しく 2 つ（0.2）増えるように修正する。

## 3. 対象範囲

- `js/logic/game.js` 内の `recordAtBatResult` 関数。

## 4. 期待される挙動

- 「ゴロ」「フライ」「ライナー」「三振」「犠打」「犠飛」の入力：1アウト（0.1）加算。
- 「併殺」の入力：2アウト（0.2）加算。
