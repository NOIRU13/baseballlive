# データベース修正計画

## 概要

チーム名と投手情報を永続化するため、MySQLデータベースとAPIサーバーを起動し、フロントエンドがAPIを利用してデータを保存できるようにする。
また、`script.js` 内で投手名入力を処理するコードにバグがあるため、これを修正する。

## 手順

1.  **Docker環境の起動**
    - `docker-compose up -d` を実行し、MySQLとAPIサーバーをバックグラウンドで起動する。
    - 起動状態を確認する。

2.  **`script.js` の修正**
    - `setupEventListeners` 内で定義されている `handleLineupInput` 関数が、外側の正しい関数を隠蔽してしまっている問題を修正する。
    - 重複している `handleLineupInput` 定義を削除し、適切な引数で呼び出せるようにする。
    - 投手入力欄 (`lineup-input-pitcher`) へのイベントリスナーが正しく機能するようにする。

3.  **動作確認**
    - ブラウザで管理画面 (`admin.html`) を開き、チーム名と投手名を入力する。
    - リロードしてもデータが保持されていることを確認する。
    - Dockerコンテナのログを確認し、APIリクエストが成功していることを確認する。
