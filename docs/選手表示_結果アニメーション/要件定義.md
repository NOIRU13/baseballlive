# 要件定義書: 打者・投手表示および結果アニメーション

## 1. 背景・目的

野球の試合配信において、視聴者が現在の対戦状況（誰が投げ、誰が打っているか）を一目で把握できるようにしたい。
また、ヒットやホームランなどの重要なプレイが発生した際に、視覚的なインパクトを与えることで、配信の盛り上がりを演出したい。

## 2. 機能要件

### 2.1. 打者・投手表示機能

- **打者表示 (Batter)**
  - 現在の打者名を大きく表示すること。
  - 打者の当日の全打席結果（例: 遊ゴロ、左安...）を表示すること。
  - 表示位置: 画面左下。
- **投手表示 (Pitcher)**
  - 現在の投手名を大きく表示すること。
  - 表示位置: 画面右下。

### 2.2. 結果アニメーション機能

- **トリガー**
  - 管理画面で打席結果（単打、二塁打、三塁打、本塁打、四球、死球、エラー、アウト各種、犠打、FC、併殺）が確定されたタイミング。
- **表示内容**
  - 結果に対応するテキスト（例: "SINGLE", "HOMERUN", "OUT"）を画面中央に表示。
- **アニメーション動作**
  - フェードインしながら出現・拡大し、数秒間滞在した後、フェードアウトして消えること。
  - 結果の種類に応じて色やスタイルを区別することが望ましい（例: 安打系＝赤/金、アウト系＝青/グレー）。

## 3. 非機能要件

- **デザイン**
  - 既存のスコアボードのデザイン（黒背景、クリムゾンレッド、ゴールド）と調和すること。
- **パフォーマンス**
  - アニメーション表示中もスコアボードの描画や更新がスムーズに行われること。
- **運用**
  - OBSブラウザソースとして使用するため、1920x1080の解像度で適切に表示されること。
