# CSSリファクタリング

style.css の肥大化に伴い、保守性を向上させるため、機能ごとにCSSファイルを分割する。
既存の `style.css` は、分割した各CSSファイルを `@import` するエントリーポイントとして機能させる。

## ファイル構成案

- **style.css**: エントリーポイント（各CSSの読み込み）
- **css/variables.css**: テーマカラー、フォントなどのグローバル変数
- **css/core.css**: リセットCSS、bodyの基本スタイル、ユーティリティクラス
- **css/layout.css**: 全体のレイアウト（#broadcast-layout）、Grid/Flexなどの大枠
- **css/components.css**: ボタン、入力フォーム、共通UIパーツ
- **css/scoreboard.css**: スコアボード、カウント、ランナー、イニング表示など
- **css/player.css**: 選手リスト、ラインナップ、選手詳細表示
- **css/animation.css**: アニメーション、トランジション定義
- **css/admin.css**: 管理画面特有のスタイル（`body.admin-mode` 以下など）

## 移行手順

1. `css` ディレクトリを作成する。
2. 上記の各ファイルを作成し、`style.css` の該当部分を移動する。
3. `style.css` を、各ファイルを `@import` する形に書き換える。
4. 動作確認を行う。
