# 実装計画書: スコアボード位置変更およびイニング表示

## 概要

現在の画面下部にあるスコアボード（コントロールバー）画面上部に移動させる。
また、現在は合計得点のみの表示だが、1回～9回までのイニングごとの得点（ランニングスコア）が表示されるように拡張する。

## ユーザーレビューが必要な項目

- 特になし（デザインは既存踏襲）

## 変更内容

### HTML (`index.html`)

- `.bottom-control` クラスを `.top-scoreboard` にリネームまたは置換。
- 内部の `.match-score` を削除し、ランニングスコア用のHTML構造（テーブルまたはグリッド）を追加。
  - チーム名列
  - 1～9回のスコア列
  - R, H, E の合計列

### CSS (`style.css`)

- `.top-scoreboard` クラスを追加。配置を `top: 40px` に設定。
- 幅を広げ、詳細なスコア表が収まるように調整。
- ランニングスコア用のテーブルスタイル定義（ボーダー、文字サイズ、色など）。

### JavaScript (`script.js`)

- `updateScoreDisplay` 関数を改修。
  - `state.scores` 配列に基づき、各イニングのセル (`#score-away-1` 等) に数値をセットする処理を追加。
- 既存の合計点やRHEの更新処理も維持・統合。

## 検証計画

### 自動テスト

- 特になし（UI変更のため）

### 手動検証

1. **ブラウザで確認**:
   - `index.html` を開き、スコアボードが画面上部に表示されていることを確認する。
   - ランニングスコア表（1-9回、RHE）が表示されていることを確認する。
2. **動作確認**:
   - `admin.html` で「イニング操作（進む・戻る）」「得点操作（＋・－）」を行い、正しいイニングのセルに得点が反映されることを確認する。
   - 合計得点（R）およびヒット（H）、エラー（E）が正しく連動して更新されることを確認する。
