# 実装計画書: スコアボード位置変更およびイニング表示

## 概要

現在の画面下部にあるスコアボード（コントロールバー）画面上部に移動させる。
また、現在は合計得点のみの表示だが、1回～9回までのイニングごとの得点（ランニングスコア）が表示されるように拡張する。

## 変更対象ファイル

- `index.html`: スコアボードのHTML構造をランニングスコア形式（テーブルまたはグリッド）に変更。
- `style.css`: スコアボードを上部に配置し、横長のランニングスコアが収まるようにスタイル調整。
- `script.js`: 各イニングのスコアをHTMLに反映するロジックを追加。

## 詳細仕様

### 1. レイアウト変更 (Top Alignment)

- 現在の `.bottom-control` を `.top-scoreboard` に変更（意味合い的にも）。
- `bottom: 30px` を `top: 40px` に変更。
- 幅を広げ（1200px -> 1600px程度）、詳細情報が入るスペースを確保。

### 2. ランニングスコア表示 (Line Score)

- チーム名、1～9回のスコア、R（合計得点）、H（安打）、E（エラー）を表形式で表示。
- **構成案**:
  | Team | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 | R | H | E |
  | --- | - | - | - | - | - | - | - | - | - | - | - | - |
  | Away | 0 | 0 | 1 | ... | | | | | | 1 | 2 | 0 |
  | Home | 0 | 2 | 0 | ... | | | | | | 2 | 3 | 0 |

- これまでの「イニング数表示」「カウント(BSO)」「ランナー」もこの上部バー内に統合するか、あるいは少し配置を調整してバランスを取る。
  - **統合案**:
    - 中央: ランニングスコア
    - 左側: イニング数・カウント・ランナー（コンパクトに）
    - あるいは、ランニングスコアの下にカウント類をぶら下げる。
  - **採用案**:
    - 上部帯に「ランニングスコア」をメインで配置。
    - その左右または下に「カウント・ランナー」などの情報を配置するデザインとする。
    - 今回は、既存の `.bottom-control` の中身を維持しつつ、真ん中の「VSスコア」部分を「ランニングスコア」に置き換える形をとる。

### 3. JavaScriptロジック

- `updateScoreDisplay` 関数を改修。
- `state.scores` 配列をループし、HTMLの対応するセル（例: `#score-away-1`, `#score-away-2`...）に数値を埋め込む。
- 現在のイニングまでのスコアを表示し、未到達回は空白またはハイフン、0などを表示（要件に合わせて調整）。

## 実装ステップ

1.  **CSS調整 (`style.css`)**
    - `.bottom-control` クラスをベースに、上部配置用のスタイル `.top-scoreboard` を作成。
    - ランニングスコア用のテーブルスタイル定義。

2.  **HTML構造変更 (`index.html`)**
    - `.bottom-control` を `div id="scoreboard-area"` などにリネームし、上部へ移動。
    - 内部の `.match-score` を削除し、ランニングスコア用のテーブル構造を挿入。

3.  **JS実装 (`script.js`)**
    - `updateScoreDisplay` 関数内で、ランニングスコアの各セル (`td`) に対して値を設定する処理を追加。

## 注意点

- 既存の「打者・投手表示（左下・右下）」はそのままの位置（下部）に残す。
- アニメーションオーバーレイは中央のまま。
