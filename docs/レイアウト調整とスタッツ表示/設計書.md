# レイアウト調整とスタッツ表示 設計書

## 概要

ユーザーの要望に基づき、以下の改修を行う。

1. `index.html` の画面サイズを 1980x1080px (または 1920x1080px) に固定。
2. スコアボードのイニング表示を全9イニング分表示するように拡張。
3. 画面下部に現在の打者と投手の詳細情報を表示するエリアを追加。

## 仕様詳細

### 1. 画面サイズ固定

- **サイズ**: リクエストは「1980x1080px」だが、一般的なフルHDは「1920x1080px」。ユーザーの要望により 1980x1080px として設定。
- **実装方針**: `body` またはルートコンテナに `width: 1980px; height: 1080px;` を適用し、画面全体を固定。

### 2. スコアボード表示拡張

- 現在の「合計点のみ」の表示から、**1回〜9回 + 合計 + R/H/E** の形式に変更。
- **レイアウト**:
  - 行: 先攻(Away), 後攻(Home)
  - 列: チーム名, 1, 2, 3, 4, 5, 6, 7, 8, 9, R, H, E
- **実装方針**: Grid または Table レイアウトを使用。JS側でループ生成。

### 3. 下部スタッツ表示

画面下部に左右に分かれた情報ボックスを配置。

#### 左側：現在の打者 (Batter)

- **表示項目**:
  - 名前
  - 今日の打席結果（例：右安、空三振、四球）
- **データ取得**:
  - 名前: `state.lineup[team][currentBatter]`
  - 結果: `state.atBatResults[team][currentBatter]` の配列データをラベル変換して表示。

#### 右側：現在の投手 (Pitcher)

- **表示項目**:
  - 名前
  - 今日の奪三振数 (K)
  - 投球回 (IP)
  - 失点 (Runs)
- **データ取得**:
  - 名前: `state.positions[defensePositions]` で守備位置が「投」(Pitcher) の選手のインデックスを探し、`state.lineup` から名前を取得。
  - 奪三振 (K): 相手チームの打席結果履歴 `state.atBatResults[offense]` を走査し、`strikeout` の数をカウント。
  - 投球回 (IP): `(現在のイニング - 1) + (現在のアウトカウント / 3)`。
    - 表記: `3` または `3 1/3` または `3 2/3`。
  - 失点 (Runs): `state.stats[defense].r` (相手チームの総得点)。
